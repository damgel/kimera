¿Qué es Hibernate Kimera?

Hibernate Kimera es una libreria muy ligera para realizar la conexión de la capa de negocios con la capa de acceso a datos, aplicando el patron fachada para ocultar toda la complejidad inherente a la capa de acceso a datos, ofreciendo una interfaz simple y amigable al desarrollador

¿Qué ofrece de diferente?

Hibernate Kimera esta pensado para utilizar la menor cantidad de líneas de código a la hora de acceder a los datos de una base de datos relacional. Posee un único objeto (Kimera) que contiene todos los métodos necesarios para obtener, persistir, actualizar o remover registros de la base de datos (independientemente de la tabla) de manera sencilla. Para esto utiliza Tipos genéricos, con lo cual basta con indicar la clase del objeto mapeado en Hibernate para poder realizar las operaciones antes mencionadas.

Un ejemplo sencillo

Supongamos que tenemos una tienda de mascotas, en la misma existen mascotas, empleados, accesorios, jaulas, etc. Nuestra base de datos tendra minimamente las tablas "mascota","empleados",etc. Primeramente mapeamos la base de datos utilizando Hibernate Tools (creamos los archivos hibernate.cfg.xml, hibernate.revenge.xml y los correspondientes POJO's) y ya estamos en condiciones de utilizar Hibernate Kimera.

Kimera k = new Kimera();
Mascota gato = k.byID("nombre", "Tom", Mascota.class); //Toda mascota tiene un nombre, existe en la tabla Mascota una mascota con nombre = "Tom"
Empleado emp = k.byID("dni","34093153", Empleado.class); //Obtiene el empleado con DNI=34093153 de la tabla empleado
emp.setDni("31526874");
k.update(emp); //Actualiza el dni del empleado 
List<Accesorio> accesorios = k.all(Accesorio.class); //Obtiene todos los accesorios de la tabla accesorio
Mascota bethoven = new Mascota();
bethoven.setNombre("Bethoven");
bethoven.setRaza("Perro");
k.add(bethoven); //Agrega una nueva mascota a la tabla Mascota

Como se puede apreciar el objeto Kimera infiere los tipos de objetos devueltos a partir de la clase pasada como parametro, esto posibilita que la libreria ofrezca una flexibilidad y reutilización muy alta.
Las combinaciones son ilimitadas, solo deja volar tu imaginación y Hibernate Kimera te ayudará a realizarlas.

¿Puedo llamar a procedimientos almacenados?

Si. Hibernate Kimera te proporciona un método de bajo nivel para realizar peticiones en SQL . Por ejemplo

Kimera k = new Kimera();
String query = "SELECT * FROM mascota m WHERE m.nombre = :nombre"; 
Map<String, Object> params = new HashMap<>();
params.put("nombre","Tom");
List result = k.callProcedure(query,params); //Obtiene una lista de objetos tipo Object con todas las mascotas con nombre "Tom"
